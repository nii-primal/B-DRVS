## Phase 1 Setup — Step by Step

### Prerequisites
| Tool | Min Version | Check |
|------|-------------|-------|
| Docker | 24.x | `docker --version` |
| Docker Compose v2 | 2.x | `docker compose version` |
| Go | 1.21+ | `go version` |
| Git | any | `git --version` |

---

### Step 0 — Clone and enter the repo
```bash
git clone https://github.com/<your-org>/bdrvs.git
cd bdrvs/bdrvs-network
```

### Step 1 — Install Fabric binaries (one-time only)
```bash
chmod +x scripts/install-fabric.sh
./scripts/install-fabric.sh

# Reload PATH
source ~/.bashrc

# Verify
cryptogen version
configtxgen --version
peer version
```

### Step 2 — Bring the network up
```bash
chmod +x scripts/network.sh
./scripts/network.sh up
```
This runs:
1. `cryptogen` → generates TLS certs + MSP for Orderer, MoH, NITA
2. `configtxgen` → creates genesis block + channel transaction artefacts
3. `docker compose up` → starts orderer + both peers + CLI admin shell

### Step 3 — Create the channel
```bash
./scripts/network.sh channel
```
This:
- Creates **bdrvschannel**
- Joins `peer0.moh.bdrvs.gh` and `peer0.nita.bdrvs.gh`
- Sets anchor peers for gossip

### Step 4 — Verify everything is running
```bash
./scripts/network.sh status

# Inspect the channel from inside the CLI container
docker exec bdrvs_cli peer channel list
docker exec bdrvs_cli peer channel getinfo -c bdrvschannel
```

```
┌───────────────────────────────────────────┐
│          bdrvs_net (Docker bridge)        │
│                                           │
│  ┌─────────────────────────────────────┐  │
│  │ GOVERNMENT BLOCKCHAIN NETWORK       │  │
│  │                                     │  │
│  │  [orderer.bdrvs.gh:7050]            │  │
│  │          ↓ orders blocks            │  │
│  │  [peer0.moh.bdrvs.gh:7051]          │  │
│  │       MoHMSP — endorser             │  │
│  │  [peer0.nita.bdrvs.gh:9051]         │  │
│  │       NITAMSP — endorser            │  │
│  │                                     │  │
│  │  Both peers must endorse every tx   │  │
│  └─────────────────────────────────────┘  │
└───────────────────────────────────────────┘
```
